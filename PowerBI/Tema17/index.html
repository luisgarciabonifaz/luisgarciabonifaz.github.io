    21. Introducción a la Interfaz

    A lo largo de este tema, se estudiará el Servicio de Power BI. Power BI Desktop es un complemento que permite diseñar Dashboards localmente. El Servicio Power BI, accesible desde cualquier navegador, facilita la presentación de datos mediante la publicación de informes.

    31. Introducción a la Interfaz

    El Servicio Power BI es el núcleo, accesible desde cualquier navegador. Aquí se pueden crear paneles a partir de informes diseñados en Power BI Desktop.

    41. Introducción a la Interfaz

    En Power BI Service, se pueden crear reportes, siendo común la publicación de informes desarrollados desde Power BI Desktop, formando el quinto paso del ciclo de desarrollo y publicación de Dashboards (Data Reporting - DR).

    51. Introducción a la Interfaz

    Conceptos básicos de la interfaz: conjuntos de datos, informes, paneles, áreas de trabajo.

    61. Introducción a la Interfaz

    Conjuntos de datos: Fuentes de datos que alimentan informes y paneles.

    71. Introducción a la Interfaz

    Informes: Conjuntos analíticos de visualizaciones asociados a un conjunto de datos.

    81. Introducción a la Interfaz

    Paneles (Dashboards): Conjuntos de visualizaciones extraídas de uno o más informes, alimentándose de varios conjuntos de datos.

    91. Introducción a la Interfaz

    Áreas de trabajo: Contenedores para organizar paneles, informes, conjuntos de datos, etc. Permiten tener áreas específicas para marketing, finanzas, etc.

    101. Introducción a la Interfaz

    Resumiendo el comportamiento: Seleccionar un área de trabajo para gestionar informes. "Mi área de trabajo" muestra el contenido de esa área; seleccionar otra área muestra el contenido correspondiente.

    111. Introducción a la Interfaz

    Lista de áreas de trabajo creadas con contenido asignado. "Mi área de trabajo" es el área por defecto.

    121. Introducción a la Interfaz

    Una vez creado un panel, se puede analizar datos, fijar plazos de actualización y compartir información con otros profesionales.

    142. Áreas de Trabajo

    Componentes detallados para comprender mejor cada parte del servicio.

    152. Áreas de Trabajo

    Áreas de trabajo: Lugares de colaboración para incluir colecciones de paneles e informes.

    162. Áreas de Trabajo

    Los diseñadores pueden compartir áreas de trabajo con compañeros y agrupar colecciones de paneles e informes en aplicaciones para distribuirlas a la comunidad.

    172. Áreas de Trabajo

    "Mi área de trabajo" es el espacio personal de cada usuario, donde pueden crear contenido.

    182. Áreas de Trabajo

    Áreas de trabajo incluyen el contenido de una aplicación específica, que puede contener paneles, informes y conjuntos de datos.

    192. Áreas de Trabajo

    Dentro de las áreas de trabajo se configuran permisos de acceso y tipo de acceso para restringir el acceso a usuarios específicos.

    202. Áreas de Trabajo

    Al conceder acceso, se asignan personas o grupos a roles de área de trabajo: Visor, Miembro, Colaborador o Administrador.

    212. Áreas de Trabajo

    Existen cuatro tipos de roles para usuarios: Visor, Colaborador, Miembro y Administrador.

    263. Informes

    Informes en Power BI representan la visualización de un cuadro de mando creado en Power BI Desktop, publicado para que otros usuarios tengan acceso a la representación de los datos.

    272. Áreas de Trabajo

    Un informe es una vista de perspectivas de un conjunto de datos con objetos visuales que representan indicadores sobre los datos. Pueden tener una o varias páginas llenas de objetos visuales.

    282. Áreas de Trabajo

    Ejemplo de informe con varias páginas de navegación y diferentes objetos visuales que forman paneles.

    292. Áreas de Trabajo

    Power BI basa el informe en conjuntos de datos. Los objetos visuales son interactivos, al igual que en la versión Desktop.

    303. Informes

    La interacción con el contenido (filtrado, segmentación, etc.) no altera el conjunto de datos subyacente ni el contenido original compartido.

    313. Informes

    Un informe consta de una o varias páginas de objetos visuales. Se comparten directamente con usuarios o como parte de una aplicación, requiriendo una licencia Power BI Pro o formar parte de un área de trabajo almacenada en una capacidad Premium.

    323. Informes

    Para cargar un informe, se accede desde el área de trabajo o la carpeta "Compartido conmigo", donde aparecerán elementos compartidos con el usuario.

    333. Informes

    Navegación entre páginas similar a Power BI Desktop. Se pueden ver en el explorador de páginas en la barra vertical izquierda.

    343. Informes

    Ajustar la forma en que se muestra una página del informe para adaptarse a la pantalla donde se visualiza.

    353. Informes

    Opciones de visualización para adaptar al ancho de la página en diferentes dispositivos. Destaca la opción de "Colores de contraste alto" para mejorar la accesibilidad.

    444. Libros de Trabajo

    Los libros de trabajo en Power BI son modelos de datos con varias tablas de datos cargadas, admitiendo cuatro tipos diferentes.

    454. Libros de Trabajo

    Tipos de libros de trabajo: Rangos de libros de trabajo, Modelos de datos en libros de trabajo, Conexiones de fuentes de datos externas con libros de trabajo, Libros de trabajo de Power BI con gráficos, tablas dinámicas y hojas de vista.

    464. Libros de Trabajo

    En resumen, un libro de trabajo es un archivo con extension .xlsx, creado con Power BI Desktop. Se pueden abrir y editar con Excel para Power BI.

    474. Libros de Trabajo

    Diferencias entre un libro de trabajo de Power BI y un archivo Excel tradicional: en un libro de trabajo de Power BI, se pueden realizar más acciones, como filtrar datos y crear informes.

    484. Libros de Trabajo

    En Power BI, se pueden usar archivos de Excel tradicionales, pero la funcionalidad es limitada. La creación de informes y la visualización avanzada solo están disponibles en Power BI Desktop.

    494. Libros de Trabajo

    Los libros de trabajo se pueden compartir de diversas maneras: enviando el archivo .xlsx, publicándolo en Power BI Service o almacenándolo en OneDrive para la Empresa.

    566. Gestión de los Conjuntos de Datos

    Los conjuntos de datos en Power BI se crean a partir de archivos de Excel, bases de datos en línea, servicios web y otros orígenes.

    576. Conjuntos de Datos

    Los conjuntos de datos son la base para informes y paneles. Al crear un informe, se selecciona un conjunto de datos para analizar.

    586. Conjuntos de Datos

    Conjuntos de datos se pueden actualizar según un horario predefinido o manualmente para obtener datos en tiempo real.

    596. Conjuntos de Datos

    El programador de actualización automática permite establecer la frecuencia de actualización del conjunto de datos.

    606. Conjuntos de Datos

    La sección de configuración del conjunto de datos en Power BI Service muestra opciones como campos, relaciones, medidas y datos ocultos.

    616. Conjuntos de Datos

    Se pueden realizar tareas de limpieza y transformación de datos en Power BI Service.

    677. Gestión de Usuarios

    Gestión de usuarios en Power BI, asignación de roles y permisos para colaboración efectiva en áreas de trabajo compartidas.

    687. Gestión de Usuarios

    La colaboración efectiva en Power BI depende de la gestión adecuada de usuarios. En áreas de trabajo compartidas, se deben asignar roles y permisos.

    697. Gestión de Usuarios

    Roles comunes: Visor, Miembro y Colaborador. Administrador tiene acceso total.

    707. Gestión de Usuarios

    Los roles determinan qué acciones puede realizar un usuario en un área de trabajo. Se pueden cambiar roles según sea necesario.

    717. Gestión de Usuarios

    El propietario de un área de trabajo es un Administrador y puede asignar roles a otros usuarios.

    727. Gestión de Usuarios

    Para colaborar efectivamente, se debe entender el propósito y las expectativas de cada área de trabajo.

    778. Gestión de Roles y Permisos

    Los roles y permisos en Power BI son cruciales para la seguridad y la colaboración efectiva.

    788. Gestión de Roles y Permisos

    La asignación adecuada de roles garantiza que cada usuario tenga el nivel correcto de acceso y capacidad de edición.

    798. Gestión de Roles y Permisos

    Ejemplo de roles y permisos: un "Visor" puede ver informes y paneles, pero no puede editarlos ni compartirlos.

    808. Gestión de Roles y Permisos

    El "Colaborador" puede crear y editar informes y paneles, pero no puede asignar permisos.

    818. Gestión de Roles y Permisos

    El "Miembro" tiene control total sobre informes y paneles, pero no puede administrar roles.

    828. Gestión de Roles y Permisos

    El "Administrador" tiene acceso total a todas las funciones de Power BI.

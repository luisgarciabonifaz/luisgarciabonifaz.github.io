    21. Función CALCULATE

    En este capítulo, se hablará de la función CALCULATE, una parte esencial de DAX que merece un capítulo dedicado. Se profundizará en varios aspectos relacionados con CALCULATE para comprenderla completamente.

    31. Función CALCULATE

    CALCULATE es la función más poderosa de DAX, encargada de evaluar una expresión en un contexto modificado por los filtros especificados en la función. Es crucial para muchas fórmulas en informes.

    41. Función CALCULATE

    Con CALCULATE, se pueden realizar acciones como añadir nuevos filtros o modificar el contexto existente. La sintaxis de CALCULATE es simple: CALCULATE(<expresión>;<filtro1>;<filtro2>...).

    51. Función CALCULATE

    La función CALCULATE se compone de dos partes: la expresión a calcular y una serie de filtros que definen el nuevo contexto modificado por CALCULATE en el que se evaluará la expresión.

    61. Función CALCULATE

    En la sintaxis de CALCULATE, la única parte obligatoria es la expresión, lo que permite expresar una función CALCULATE sin ningún filtro y que funcione sin problemas.

    71. Función CALCULATE

    Existe una estrecha relación entre la función CALCULATE, el contexto de evaluación y el contexto de transición. Comprender estos conceptos es esencial para entender CALCULATE correctamente.

    81. Función CALCULATE

    Dentro de este capítulo, se detallará cada uno de estos conceptos, y es crucial revisarlos varias veces para comprender correctamente el funcionamiento de la función CALCULATE.

Función CALCULATE CALCULATE realiza un proceso en el que analiza el contexto de evaluación, anula o agrega filtros según sea necesario y luego ejecuta el cálculo de la expresión.

    1. Función CALCULATE El primer paso, analizar el contexto de evaluación, es común a todas las funciones en DAX. Se incorpora el contexto de evaluación para el cálculo.

    2. Función CALCULATE En el paso 2, CALCULATE demuestra su potencial al incorporar sus filtros al contexto, anulando los filtros existentes según sea necesario.

    3. Función CALCULATE Es crucial recordar la secuencia del proceso como base para entender y diseñar nuevas fórmulas.

    4. Función CALCULATE CALCULATE, al igual que otras funciones y expresiones, puede usarse en medidas o columnas calculadas, pero su comportamiento puede variar según el contexto.

    5. Función CALCULATE En una expresión de columna calculada, combinada con una expresión SUM, devuelve la suma de un campo en el contexto de filtro actual.

    Capítulo 2 – Tablas Expandidas 222. Tablas Expandidas Una tabla expandida contiene todas las columnas de la tabla base y de las tablas vinculadas a través de relaciones.

    1. Tablas Expandidas Partiendo de tablas relacionadas, se puede expandir el contenido de una tabla con el contenido de otra simplemente referenciándola.

    markdown Copy code Evaluacion = RELATED('Tabla Evaluacion'[Evaluación]) 242. Tablas Expandidas Desde la 'Tabla Empleados', utilizando la función RELATED, se puede expandir la tabla de origen con la referenciada.

    1. Tablas Expandidas La expansión de la tabla no tiene nada que ver con el filtrado bidireccional. Si se activa el filtro cruzado bidireccional en una relación, no afectará la expansión de la tabla.

    2. Tablas Expandidas Cada tabla expandida contiene columnas nativas y relacionadas. Las columnas nativas son las de la tabla original, y las relacionadas son de las tablas vinculadas.

    3. Tablas Expandidas La tabla expandida no ocurre físicamente. El motor VertiPaq solo almacena tablas nativas, pero toda la semántica de DAX se basa en el concepto teórico de tablas expandidas.

    4. Tablas Expandidas Lo generado es una tabla con parte real y parte virtual. Las columnas nativas son parte de la tabla física de origen, mientras que las columnas "virtuales" son solo una expansión sobre la tabla original.

    Capítulo 3 – Transición de Contexto 323. Transición de Contexto La transición de contexto mueve un contexto de fila a un contexto de filtro mediante dos operaciones: invalidar el contexto de fila existente y agregar columnas y valores como argumentos de filtro.

    1. Transición de Contexto La transición de contexto se utiliza para aplicar filtros basados en columnas que no están presentes en la tabla original.

    markdown Copy code Ventas = CALCULATE( SUM('Tabla Ventas'[Monto]), 'Tabla Ventas'[Fecha] = 'Tabla Calendario'[Fecha], 'Tabla Calendario'[Año] = 2022 ) 343. Transición de Contexto En el ejemplo anterior, se transiciona del contexto de fila en 'Tabla Ventas' al contexto de filtro en 'Tabla Calendario' utilizando las columnas 'Fecha' y 'Año'.

    1. Transición de Contexto La transición de contexto es fundamental para trabajar con tablas expandidas, ya que permite aplicar filtros basados en columnas de tablas relacionadas.

    2. Transición de Contexto La transición de contexto se realiza en el orden de las operaciones, primero se invalida el contexto de fila y luego se aplican los filtros.

    3. Transición de Contexto La transición de contexto es especialmente útil cuando se trabaja con tablas expandidas, permitiendo aplicar filtros basados en columnas de tablas relacionadas.

    4. Transición de Contexto Al transicionar de un contexto de fila a un contexto de filtro, se deben especificar las columnas y valores necesarios para realizar la transición.

    Capítulo 4 – Orden de Evaluación 424. Orden de Evaluación La función CALCULATE evalúa la expresión y los filtros en un orden específico. Comprender este orden es clave para diseñar fórmulas efectivas.

    1. Orden de Evaluación La expresión en CALCULATE es el último parámetro evaluado, después de todos los filtros y modificadores.

    markdown Copy code Ventas = CALCULATE( 'Tabla Ventas'[Monto], 'Tabla Calendario'[Año] = 2022, 'Tabla Ventas'[Fecha] = 'Tabla Calendario'[Fecha] ) 444. Orden de Evaluación En este ejemplo, 'Tabla Ventas'[Monto] se evalúa después de que se aplican los filtros en 'Tabla Calendario'[Año] y 'Tabla Ventas'[Fecha].

    1. Orden de Evaluación El orden de evaluación es esencial para entender y diseñar fórmulas correctamente.

    2. Orden de Evaluación Es importante destacar que si la expresión hace referencia a una medida, esta medida se evalúa en el contexto final después de aplicar todos los filtros y modificadores.

    3. Orden de Evaluación El orden de evaluación es de izquierda a derecha, primero se aplican los modificadores y filtros de izquierda a derecha y luego se evalúa la expresión.

    Capítulo 5 – Modificadores Comunes 505. Modificadores de Arquitectura de Relaciones En este capítulo, se explorarán los modificadores comunes de la arquitectura de relaciones, como USERELATIONSHIP y CROSSFILTER.

    1. Modificadores de Arquitectura de Relaciones USERELATIONSHIP se utiliza para activar temporalmente una relación específica al realizar un cálculo.

    markdown Copy code Ventas = CALCULATE( SUM('Tabla Ventas'[Monto]), USERELATIONSHIP('Tabla Ventas'[Fecha], 'Tabla Calendario'[Fecha]) ) 525. Modificadores de Arquitectura de Relaciones En este ejemplo, USERELATIONSHIP activa temporalmente la relación entre 'Tabla Ventas' y 'Tabla Calendario' basada en la columna 'Fecha'.

    1. Modificadores de Arquitectura de Relaciones CROSSFILTER modifica la dirección de la relación. Puede ser None, Single, o Both.

    2. Modificadores de Arquitectura de Relaciones CROSSFILTER se utiliza para modificar el filtro cruzado en una relación. Puede ser útil cuando se necesitan reglas de filtro específicas.

    3. Modificadores de Estructura del Contexto de Filtro Existen modificadores que remueven y restauran filtros en el contexto de CALCULATE.

    4. Modificadores de Estructura del Contexto de Filtro ALL, ALLEXCEPT y ALLNOBLANK remueven filtros del contexto, permitiendo realizar cálculos ignorando ciertos filtros.

    markdown Copy code Ventas = CALCULATE( SUM('Tabla Ventas'[Monto]), ALL('Tabla Calendario'[Mes]) ) 575. Modificadores de Estructura del Contexto de Filtro En este ejemplo, ALL remueve el filtro aplicado a 'Tabla Calendario'[Mes], permitiendo sumar todas las ventas independientemente del mes.

    1. Modificadores de Estructura del Contexto de Filtro REMOVEFILTERS es un alias de ALL y remueve todos los filtros del contexto, devolviendo todas las filas sin ningún filtro aplicado.

    2. Modificadores de Estructura del Contexto de Filtro Existen modificadores que restauran filtros en el contexto de CALCULATE.

    3. Modificadores de Estructura del Contexto de Filtro ALLSELECTED restaura los filtros aplicados por objetos visuales externos.

    markdown Copy code Ventas = CALCULATE( SUM('Tabla Ventas'[Monto]), ALLSELECTED('Tabla Calendario') ) 616. Modificadores de Estructura del Contexto de Filtro En este ejemplo, ALLSELECTED restaura los filtros aplicados a 'Tabla Calendario' por objetos visuales externos.

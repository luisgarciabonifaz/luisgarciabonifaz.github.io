    Capítulo 1 – Formateo de código

    1.1 Formateo de código

    Existen reglas para el formateo de la estructura de las funciones desarrolladas. Esto es parte de las buenas prácticas. A continuación se detallan algunas de estas buenas prácticas para mantener una estructura estandarizada en las funciones.

    1.2 Ejemplo de Formateo de código

    Un ejemplo de medida DAX escrita en una sola línea:

    ClientesEjerActual = CALCULATE ( DISTINCTCOUNT ( Ventas[CodigoCliente] ); FILTER ( Ventas; YEAR ( Ventas[FechaFactura] ) = MAX ( Calendario[Año] ) ) )

    1.3 Pasos para el Formateo de código

    1. Escribir un espacio entre cada paréntesis de apertura y cierre.
    2. Colocar un espacio antes de cada operador u operando.
    3. Si la función está escrita en varias líneas, el paréntesis de apertura "(" estará en la misma línea de llamada a la función.
    4. El paréntesis de cierre ")" se escribirá antes de empezar con los argumentos de la función.
    5. Siempre poner los argumentos de la llamada a una función en una nueva línea, con una sangría respecto a la llamada a la función.
    6. La "," que separa dos argumentos está en la misma línea que el argumento anterior.
    7. Solo escribir una función en una única línea si tiene un argumento que no es una llamada a una función.
    8. Si se llama a una función en una expresión que ya contiene varias líneas, deberá estar siempre en una nueva fila, precedida de un operador.

    1.4 Ejemplo de Uso de Comentarios

    Cuando se inicia el desarrollo de una función DAX, es importante seguir ciertos pasos para garantizar la sintaxis correcta. Por ejemplo:

    ClientesEjerActual = CALCULATE ( DISTINCTCOUNT ( Ventas[CodigoCliente] ), FILTER ( Ventas, YEAR ( Ventas[FechaFactura] ) = MAX ( Calendario[Año] ) ) )

    Capítulo 2 – Adición de comentarios en el código

    2.1 Importancia de los comentarios

    En DAX, al igual que en otros lenguajes, se pueden utilizar comentarios para detallar partes de la función o recursos utilizados, proporcionando más información a quienes necesiten utilizar esas funciones.

    2.2 Tipos de comentarios

    • Línea única de comentario: // Esto es un comentario en una única línea
    • Varias líneas de comentario: /* Esto es un comentario en varias líneas */
    • Alternativa a única línea de comentario: -- Esto sería un comentario válido.

    2.3 Ejemplo de Uso de Comentarios

    Ejemplo de cálculo con comentarios:

    DAX Sueldo Medio = // Calculo de sueldos medios CALCULATE( / Calculo realizado para sueldos superiores a 100000€ / AVERAGE('Tabla Sueldo'[Sueldo]), 'Tabla Sueldo'[Sueldo]>100000 -- Este valor filtra los salarios mayores a 100000 )

    2.4 Detalles sobre el Uso de Comentarios

    El uso de comentarios puede ser sobre una función, en la misma línea de una función, variable o expresión, o para explicar partes del contenido de la función detalladamente. Es esencial encontrar un equilibrio entre información útil y evitar saturar el código con comentarios poco relevantes.

    Capítulo 3 – Orden de evaluación

    3.1 Concepto de orden de evaluación

    El orden de evaluación en DAX es crucial para comprender cómo se evalúan los cálculos y obtener resultados precisos. Consta de dos partes: contexto de filtro y contexto de fila.

    3.2 Contexto de filtro

    El contexto de filtro combina filtros aplicados antes de la evaluación de una expresión DAX. Puede ser implícito (por filas, columnas, slicers o filtros) o explícito (en la propia expresión DAX).

    3.3 Contexto de fila

    El contexto de fila se utiliza para calcular una expresión fila por fila. Permite realizar cálculos fila por fila dentro de una tabla.

    3.4 Ejemplo de Contexto de Filtro

    Ejemplo de medida utilizando el contexto de filtro:

    Sueldos IT = CALCULATE( SUM('Tabla Sueldo'[Sueldo]), FILTER('Tabla Empleados', 'Tabla Empleados'[Departamento]="Information Technology") )

    3.5 Estrategias para el Orden de Evaluación

    Es esencial comprender cómo el contexto de filtro y el contexto de fila afectan el orden de evaluación. Algunas estrategias incluyen garantizar la carga de datos de calidad, usar funciones tolerantes a errores y utilizar funciones como IF para determinar condiciones de error.

    Capítulo 4 – Creación de un Repositorio de Medidas

    4.1 Importancia del Repositorio de Medidas

    Un repositorio de medidas es un contenedor que alberga las medidas desarrolladas a partir de las columnas y tablas del proyecto. Ayuda en la segmentación y localización de medidas según sus objetivos.

    4.2 Estructura del Repositorio de Medidas

    El repositorio de medidas puede organizarse según áreas temáticas, módulos específicos o categorías de medidas. Su estructura debe ser clara y fácil de entender visualmente.

    4.3 Ejemplo de Repositorio de Medidas

    Ejemplo de estructura básica de un repositorio de medidas:

    Repositorio de Medidas |-- Ventas | |-- Total Ventas | |-- Ventas Netas |-- Recursos Humanos | |-- Sueldos | |-- Gastos de Personal

    4.4 Ventajas del Repositorio de Medidas

    • Facilita la gestión de medidas en proyectos grandes.
    • Permite una rápida localización de medidas específicas.
    • Mejora la colaboración entre desarrolladores.

    5.1 Importancia del Manejo de Variables

    El manejo de variables en DAX permite escribir cálculos complejos y eficaces. Mejora el rendimiento, la legibilidad del código, simplifica la depuración y reduce la complejidad de las fórmulas.

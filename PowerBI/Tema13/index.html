    1. Funciones comunes y recomendadas

    En ocasiones, puede interesar realizar algunas operaciones de forma puntual para la combinación de varias tablas. A continuación, se detallan las funciones más comunes en DAX para este propósito.

    • Combinación Virtual: Explicación reservada para el siguiente capítulo.

    Tipos de Combinación

    • Left Outer Join: Toma todas las filas de A y las coincidentes de la tabla B.
    • Right Outer Join: Toma todas las filas de B y las coincidentes de la tabla A.
    • Full Outer Join: Toma todas las filas de A y todas las de B, devolviendo una tabla con todos los registros.
    • Inner Join: Toma solo las filas coincidentes de A y B.
    • Left Anti Join: Toma todas las filas de A sin coincidencias en B.
    • Right Anti Join: Toma todas las filas de B sin coincidencias en A.

    2. Función UNION

    La función UNION devuelve una tabla resultado de concatenar las filas de dos o más tablas. Su sintaxis es la siguiente: ```dax UNION(, [,]...) Consideraciones adicionales:

    Las tablas deben tener el mismo número de columnas. Los nombres de las columnas de la tabla resultante son los mismos que los de la primera tabla pasada como argumento. Las filas duplicadas aparecerán duplicadas en el resultado. Ejemplo de uso de UNION:

    dax Copy code UNION(Paises1, Paises2) 3. Función CROSSJOIN La función CROSSJOIN realiza un producto cartesiano de las tablas definidas dentro de la expresión. Su estructura es:

    dax Copy code CROSSJOIN(

    ,
    [,
    ]...) Aspectos a tener en cuenta:

    Las columnas de la tabla resultante son todas las columnas presentes en las tablas incluidas como argumentos. Los nombres de las columnas deben ser diferentes entre las tablas. 4. Función EXCEPT La función EXCEPT muestra las filas de una tabla que no se encuentran en la otra. Su estructura es:

    dax Copy code EXCEPT(table1, table2) Requisito:

    table1 y table2 deben tener el mismo número de columnas, y la tabla resultante tendrá los mismos campos. Ejemplo de uso de EXCEPT:

    dax Copy code EXCEPT(Paises1, Paises2) 5. Función ALLEXCEPT La función ALLEXCEPT se utiliza para eliminar todos los filtros de contexto aplicados a la tabla definida en la función, excepto aquellos que hagan referencia a las columnas incluidas como parámetros. Su estructura es:

    dax Copy code ALLEXCEPT(table, columna [,columna [,...]]) Parámetros:

    table: Tabla cuyos filtros se desean eliminar. columna: Nombre de la/s columna/s cuyos filtros se desean dejar. Ejemplo de uso de ALLEXCEPT:

    dax Copy code CALCULATE(SUM('Tabla Empleados'[Sueldo]), ALLEXCEPT('Tabla Empleados', 'Tabla Empleados'[Departamento])) Estas funciones son esenciales en el modelado de datos en DAX.

    Capítulo 1 – Tipos de datos existentes

    Existen diferentes tipos de datos. Dependiendo de las operaciones que se deban realizar con dichos datos, estos deben pertenecer a unos determinados tipos de datos, de forma que sean compatibles con las operaciones a realizar o con el resto de datos con los que se van a cruzar.

    Capítulo 2 – Que tipos de orígenes de datos existen

    Power BI agrupa los tipos de conectores de los que dispone para extraer la información desde los diferentes orígenes por familias.

    Archivo

    Conexión a orígenes de tipo físico, que contienen información estructurada y compresible, para su extracción desde la herramienta. Los orígenes admitidos por la herramienta son: Excel, Texto o CSV, XML, JSON, Carpeta, PDF, Parquet, Carpeta de SharePoint, etc.

    Bases de Datos

    Conexión a diferentes tipos de bases de datos. Algunos ejemplos son: SQL Server, Access, Oracle, MySQL, PostgreSQL, Sybase, Teradata, SAP HANA, Google BigQuery, etc.

    Power Platform

    Conjunto de herramientas de Microsoft integradas dentro de Office 365, que permite integrar todos los datos alojados en este entorno. Ejemplos son: Conjuntos de datos de Power BI, Flujos de datos de Power BI, Common Data Service, Dataverse, etc.

    Azure

    Permite el acceso en tiempo real a contenido desarrollado en entornos Azure. Algunos ejemplos son: Azure SQL Database, Azure Synapse Analytics, Azure Blob Storage, Azure Cosmos DB, Azure Data Lake Storage Gen2, etc.

    Servicios en Línea

    Servicios en la nube que permiten el acceso a parte de su contenido para poder importarlos a la herramienta Power Bi. Algunos ejemplos son: Lista de SharePoint Online, Microsoft Exchange Online, Dynamics 365, Google Analytics, Salesforce, GitHub, LinkedIn Sales Navigator, etc.

    Otros orígenes de datos

    Orígenes cuya naturaleza no permite agruparlos en ningún grupo de los anteriormente vistos. Algunos ejemplos son: Web, Lista de SharePoint, Fuente de OData, Active Directory, Microsoft Exchange, Archivo Hadoop (HDFS), Script de R, Script de Python, ODBC, OLE DB, etc.

    Capítulo 3 – Tipos de importación de datos

    Dependiendo del origen de datos, se permiten diferentes tipos de conexión. Estas conexiones facilitan la sincronización de los datos en diferentes formas o la importación del contenido en modo asíncrono.

    Importar

    Importa los datos contenidos en las tablas seleccionadas (conexión asíncrona). Esto significa que importa todo el dato y metadato desde el origen, sin poder hacer un filtrado previo a lo que se va a recibir.

    DirectQuery

    Importa los datos (los metadatos están en el origen) contenidos en las tablas seleccionadas y actualiza los datos de forma automática con una periodicidad definida.

    Capítulo 4 – Ficheros

    Los datos a los que se conectan las ETLs proceden de un conjunto de datos en diferentes orígenes. Para tener un conjunto de datos, primero es necesario obtener dichos datos mediante conexiones a estas fuentes de información.

    Local

    Archivo almacenado en una unidad local en el equipo o en equipo remoto de la organización. Desde Power BI se puede importar el archivo. En realidad, se crea un nuevo conjunto de datos en Power BI y, en algunos casos el modelo de datos, incluso el conjunto de datos.

    OneDrive - Business

    Una licencia OneDrive para Empresa es la manera más eficaz de mantener el trabajo en Excel, Power BI Desktop o un archivo .CSV y de mantener sincronizados el conjunto de datos, los informes y los paneles en Power BI.

    OneDrive - Personal

    Si los archivos están almacenados en este contenedor de uso personal, se dispondrá de muchas ventajas idénticas a las que se obtienen con OneDrive para la Empresa.

    SharePoint

    Guardar los archivos de Power BI Desktop en SharePoint es prácticamente igual a guardarlos en OneDrive para la Empresa. La principal diferencia estriba en cómo se conecta al archivo desde Power BI.

    Capítulo 5 – Bases de Datos SQL

    El conector a Bases de Datos SQL, permite acceder a bases de datos que contengan información estructurada, mediante SQL, que es un lenguaje estándar para el manejo de bases de datos relacionales.

    SQL como lenguaje estándar

    Para obtener información de las bases de datos se utilizan instrucciones SQL.

    Modos de importación

    Power BI permite importar o acceder a los datos mediante las instrucciones SQL con dos modos diferentes.

    Importar

    Trae todos los datos de las tablas seleccionadas al conjunto de datos de Power BI.

    Direct Query

    Accede a los datos de las tablas seleccionadas, y de esta forma los datos que están en el origen.

    Uso de instrucciones SQL

    Para obtener los datos, se utiliza el lenguaje SQL. Este lenguaje se utiliza para obtener, actualizar, insertar y borrar datos de una base de datos.

    Capítulo 6 – Páginas web - Servicios Externos

    Los conectores de páginas web, son conectores con los que Power BI se puede conectar a una página web.

    Conexión a páginas web

    Este tipo de conectores se suelen utilizar para obtener datos estructurados. Es decir, es preferible utilizar este tipo de conectores cuando se quiere importar datos que están estructurados y no se necesita ningún tipo de transformación.

    Ejemplos de servicios externos

    Algunos de los conectores son Google Analytics, Google BigQuery, Microsoft Exchange Online, Dynamics 365, SharePoint Online.

    Importancia de datos estructurados

    Para que este tipo de conectores sean útiles, los datos deben estar estructurados.

    Capítulo 7 – Open Data

    Con este tipo de conector se puede realizar una conexión similar a la realizada con los conectores de páginas web. Con esta conexión se puede realizar la importación de datos abiertos.

    Conexión similar a Páginas Web

    Se realiza de forma muy similar a la de los conectores de Páginas Web.

    Ejemplo con Open Data

    Un ejemplo de este tipo de conector es el de la base de datos Northwind.

    Modos de importación

    Este tipo de conector solo permite la importación. Es decir, se traen todos los datos de la tabla que se va a importar.

    Conclusiones del Capítulo 4

    En este capítulo hemos aprendido sobre los conectores más utilizados en Power BI y su importancia en la preparación de datos después de la carga para el análisis en Power BI.

    Gestión de Usuarios y Grupos en GNU/Linux

    1. Introducción

    La gestión de usuarios y grupos en sistemas GNU/Linux es fundamental para garantizar la seguridad y la organización del sistema operativo.

    Podemos identificar 3 perfiles de usuarios en un sistema Linux:

    • Usuario root: También llamado superusuario o administrador.
      • Este es el único con derechos suficientes para crear o eliminar a otros usuarios.
    • Usuarios especiales: También llamadas cuentas del sistema
    • Usuarios normales: Estas son las cuentas de los usuarios personales
      • Cada usuario dispone de un directorio de trabajo que se ubica generalmente en /home.

    2. Prompt de la línea de comandos:

    El prompt, también conocido como indicador de comandos o símbolo del sistema, es el texto que se muestra en la línea de comandos y que indica que el sistema está listo para recibir órdenes del usuario. En los sistemas GNU/Linux, el prompt suele incluir información útil para el usuario, como el nombre de usuario, el nombre del host y el directorio actual. Por ejemplo:

    usuario@host:~$

    Donde:

    • usuario: Nombre de usuario actual.
    • host: Nombre del host o máquina.
    • ~: Directorio actual del usuario (normalmente el directorio de inicio).
    • $: Indica que el usuario tiene permisos normales. Si el prompt termina con #, significa que el usuario tiene permisos de superusuario (root).

    3. Usuarios

    Linux tiene un sistema de identificación de usuarios, llamado UID, así también para los grupos del sistema llamado GID.

    El usuario root siempre tiene el UID 0, la gestión de cuentas del sistema suele tener una reserva de identificaciones que van del 1 al 100, y finalmente los usuarios normales que suelen comenzar a partir del 1000 en adelante.

    Al añadir un usuario también se creará un directorio en nuestro disco con el nombre de su login. Este directorio se coloca por defecto en el directorio /home excepto para root, cuyo directorio base es /root.

    La información asociada a los usuarios en un sistema Linux se guarda en el fichero /etc/passwd y las contraseñas y datos afines en /etc/shadow.

    /etc/passwd

    Este archivo contiene información sobre los usuarios del sistema, como nombres de usuario, identificadores de usuario (UID), identificadores de grupo (GID), directorios de inicio y shell por defecto.

    Formato:

    1. Nombre del usuario.
    2. Password.
    3. UID.
    4. GID.
    5. Comentarios.
    6. Directorio home o de trabajo.
    7. Shell o Intérprete de comandos

    Ejemplo:

    root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
sync:x:4:65534:sync:/bin:/bin/sync
games:x:5:60:games:/usr/games:/usr/sbin/nologin
man:x:6:12:man:/var/cache/man:/usr/sbin/nologin
lp:x:7:7:lp:/var/spool/lpd:/usr/sbin/nologin
mail:x:8:8:mail:/var/mail:/usr/sbin/nologin

    /etc/shadow

    Este archivo almacena las contraseñas encriptadas de los usuarios. Solo es accesible por el usuario root y es fundamental para la seguridad del sistema.

    Formato:

    1. Nombre de usuario
    2. Contraseña cifrada, o si no está definida dos admiraciones (!!)
    3. Días desde el 1 de enero del 1970 en que la contraseña fue cambiada por última vez.
    4. Mínimo de días antes que al usuario se le permite cambiar la contraseña.
    5. Máximo número de días que se puede usar la misma contraseña.
    6. Número de días antes que expire la contraseña que se empieza a avisar al usuario que debe cambiarla.
    7. Días después que caduque la contraseña antes que se deshabilite la cuenta. Sería el campo 3 más los días del campo 5
    8. Días desde el 1 de enero del 1970 en que la cuenta se deshabilita.
    9. Campo reservado

    Ejemplo:

    root:!:17177:0:99999:7:::
daemon:*:17002:0:99999:7:::
bin:*:17002:0:99999:7:::
sys:*:17002:0:99999:7:::
sync:*:17002:0:99999:7:::
games:*:17002:0:99999:7:::
man:*:17002:0:99999:7:::
lp:*:17002:0:99999:7:::

    Pero los comandos habituales para la gestión de usuarios son los siguientes:

    useradd

    useradd [opciones] nombre usuario

    Opciones:

    • -d: Especifica el directorio inicial del usuario
    • -s: Especifica el shell del usuario
    • -g: Especifica el grupo primario del usuario
    • -G: Especifica los grupos secundarios del usuario. Combinarlo con a para añadir grupos
    • -m: Crea el directorio de inicio del usuario
    • -e: Especifica la fecha de expiración del usuario
    • -uid: Especifica el identificador de usuario para el usuario

    Ejemplos:

    sudo useradd -d /home/juanito juan

sudo useradd -m pedro

    userdel 

    userdel [opciones] nombre usuario

    Ejemplos:

    sudo userdel juan

sudo userdel -r juan

    Borra el directorio personal del usuario con todo su contenido

    usermod

    usermod [opciones] nombre usuario

    Opciones:

    • -d: Sirve para asignar un nuevo directorio HOME al usuario.
    • -m: Mueve el contenido del antiguo directorio al nuevo
    • -g: Cambia el grupo del usuario. Se debe usar nombre o número del grupo. Debe existir el grupo previamente.
    • -l: Permite modificar el login del usuario. OJO! no cambia el nombre del directorio HOME del usuario,debería cambiarse de forma manual.

    Ejemplos:

    Para cambiar el directorio de inicio del usuario juan para que sea /home/profesores/juan. La opción -m hace que mueva el contenido del antiguo directorio al nuevo emplazamiento.

    sudo usermod -d /home/profes/juan -m juan

    Para cambiar el grupo inicial del usuario juan para que sea profesores.

    sudo usermod -g profesores juan

    Para cambiar el nombre del usuario juan. El nuevo nombre es jorge.

    sudo usermod -l jorge juan

    Para cambiar la contraseña a un usuario utilizar el comando, que nos pedirá por pantalla la nueva contraseña

    sudo passwd usuario

    4. Grupos

    En Linux al igual que en Windows también existen grupos de usuarios.

    • Los grupos permiten otorgar los mismos privilegios a un conjunto de usuarios.
    • Siempre que se añada un usuario al sistema se creará un grupo con su mismo nombre, llamado grupo primario.
    • La información asociada a los grupos en un sistema Linux se guarda en el fichero /etc/group

    /etc/group

    Contiene información sobre los grupos del sistema, incluyendo los nombres de grupo, identificadores de grupo (GID) y los usuarios que pertenecen a cada grupo.

    Formato:

    1. Nombre del grupo. El nombre del grupo.
    2. Contraseña del grupo. Ésta es opcional, pero si se fija permite que usuarios que no son parte del grupo se unan al mismo.
    3. ID del grupo (GID). El equivalente numérico del nombre del grupo.
    4. Miembros del grupo. Todos los nombres de usuario que pertenecen al grupo, en forma de lista separada por comas.

    Ejemplo:

    root:x:0:
daemon:x:1:
bin:x:2:
sys:x:3:
adm:x:4:syslog,luis
tty:x:5:
man:x:12:
proxy:x:13:
kmem:x:15:
voice:x:22:
cdrom:x:24:luis

    Los comandos habituales para la gestión de grupos son los siguientes:

    groupadd

    groupadd [opciones] nombre _grupo

    Ejemplo:

    groupadd profesores

    groupdel:

    groupdel [opciones] nombre _grupo

    Ejemplo:

    groupdel profesores

    groupmod

    groupmod [opciones] nombre _grupo

    Opciones:

    • -g : El valor numérico del identificador de grupo. Este valor debe ser único. Los valores entre 0 y 99 se reservan normalmente para cuentas del sistema. Este valor no puede ser negativo
    • -n: El nombre del grupo será cambiado de grupo a grupo

    Ejemplo:

    groupmod -n alumnos profesores

    Para añadir y borrar usuarios a un grupo utilizaremos los comandos adduser y deluser, especificando el usuario y el grupo

    adduser juan profesores  // Añadir a 'juan' al grupo 'profesores'

deluser juan profesores  // Quitar a 'juan' del grupo 'profesores'

    5. Usuario Root

    En muchas distribuciones de GNU/Linux, la cuenta del superusuario o root viene desactivada por defecto, por seguridad. De esta forma no provocaremos destrozos en el sistema por desconocimiento de lo que estamos haciendo o accidente.

    La cuenta del usuario que creamos al instalar Ubuntu no es la misma que la cuenta del root o superusuario, sino que pertenece al grupo de administradores.

    Características:

    • El usuario root es el superusuario en los sistemas UNIX y GNU/Linux.
    • Tiene privilegios de administración completos y puede realizar cualquier acción en el sistema.
    • Es importante usar el usuario root con precaución, ya que cualquier error puede tener consecuencias graves para la seguridad y estabilidad del sistema.
    • Para acceder al usuario root, se utiliza el comando su (Switch User) seguido de la contraseña de root.
    • Ejemplo: su - (para iniciar sesión como root)

    Para activar la cuenta de root.

    sudo passwd root

    Para desactivarla:

    sudo passwd -l root

    Comandos relativos al cambio de usuario:

    su

    Son las siglas de "Switch User" y sirve para cambiar de usuario sin necesidad de hacer un cierre o cambio de sesión.

    Ejemplo:

    su manuel

    Si queremos cambiar al usuario "manuel" y además cambiar a su carpeta personal y demás variables de entorno, añadimos un espacio en blanco y un guión medio " " a "su" (antes de manuel):

    su - manuel

    sudo

    "Sudo" viene de "switch user do" y permite a los usuarios ejecutar acciones con los privilegios de seguridad del root, de manera segura.

    Para administrar el sistema existe un grupo de usuarios denominado "sudoers users" (administradores o admin), los cuales pueden obtener permisos de root, mediante la utilización de sudo.

    El usuario con el que instalamos GNU/Linux, se encuentra incluido en este grupo de administradores.

    El archivo que determina el acceso a las tareas administrativas del grupo "sudoers users" es "/etc/sudoers".

    Perfiles

    Para cada usuario existen tres ficheros muy importantes. Estos ficheros o (shells scripts) se conocen como ficheros perfiles:

    • .bash_profile o .profile se ejecuta siempre que se abra una nueva sesión en el sistema.
    • .bashrc: se ejecuta siempre que se invoque un nuevo shell. Por lo general en él se colocan las definiciones de funciones y los alias de comandos.
    • .bash_history: guarda las líneas de comandos que se han ido ejecutando en el sistema.

    Cuando se crea un nuevo usuario se le colocan en su directorio todos los archivos del directorio /etc/skel. Estos ficheros son los que ajustan los perfiles para el shell.

    Aparte, existen en el home de cada usuario otros ficheros que cargan perfiles para los entornos gráficos como el gnome, perfiles para el open office, etc.

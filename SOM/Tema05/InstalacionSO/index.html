    5. Instalación Sistemas Operativos

    5.1. Estructura física del disco duro

    La estructura física se refiere a cómo están dispuestos los componentes físicos del disco duro, como las caras, pistas y sectores.

    • Caras:
      • Un disco duro consta de dos caras, siendo cada cara una superficie magnética donde se puede almacenar información.
    • Pistas:
      • Las pistas son círculos concéntricos en cada cara del disco. Cada pista representa una ubicación específica en el disco.
    • Cilindro:
      • Un cilindro es un conjunto de pistas con el mismo número en cada disco. Imagina un cilindro vertical que atraviesa todas las caras del disco.
    • Sectores o bloques físicos:
      • Los sectores o bloques físicos son la unidad mínima de almacenamiento en un disco duro. Representan la cantidad de información que se lee o escribe en una operación de lectura/escritura.

    5.2. Estructura lógica del disco duro

    La estructura lógica se centra en cómo el sistema operativo organiza y accede a la información en el disco, utilizando conceptos como clusters, FAT, directorios y la zona de datos. En conjunto, la combinación de la estructura física y lógica permite un almacenamiento eficiente y la recuperación de datos en un disco duro.

    • Clusters:
      • El sistema operativo trabaja con clusters, que son agrupaciones de sectores. Un cluster es el tamaño mínimo de un archivo y la unidad básica con la que opera el sistema de archivos.
    • Sector de Arranque (Boot):
      • El sector de arranque, ubicado en el primer sector del disco (sector 0), contiene un pequeño programa que carga el sistema operativo en la memoria cuando se inicia la computadora.
    • Tabla de Asignación de Ficheros (FAT):
      • La FAT organiza la información en forma de ficheros en la zona de datos del disco duro. Sirve como un índice que el sistema operativo utiliza para localizar archivos.
    • Directorio Raíz:
      • El directorio raíz, de tamaño fijo y situado detrás de la FAT, contiene información referente a la zona de datos y proporciona una estructura jerárquica para organizar archivos y carpetas.
    • Zona de Datos:
      • La zona de datos es donde se almacena la información real de archivos y directorios.

    5.3. Particiones

    Las particiones en un disco duro son divisiones lógicas o secciones separadas que se crean en el espacio de almacenamiento de un disco. Estas particiones permiten organizar y gestionar mejor la información almacenada en el disco.Hay varios tipos de particiones:

    • Partición Primaria:
      • Es la partición principal en un disco duro.
      • Puedes tener hasta cuatro particiones primarias en un disco.
      • Una de estas particiones primarias puede ser designada como "activa" y contener el cargador de arranque del sistema operativo.
    • Partición Extendida:
      • Se utiliza para crear particiones lógicas adicionales dentro de ella.
      • Puedes tener solo una partición extendida en un disco.
      • No se utiliza directamente para almacenar datos, pero sirve como contenedor para particiones lógicas.
    • Partición Lógica:
      • Se crea dentro de una partición extendida.
      • No puedes arrancar un sistema operativo directamente desde una partición lógica; necesitas una partición primaria o una partición activa para eso. -Puedes tener varias particiones lógicas dentro de una partición extendida.

    Una tabla de particiones es una estructura de datos en un disco duro que contiene información sobre la organización de las particiones en ese disco. En términos más simples, la tabla de particiones actúa como un mapa que le dice al sistema operativo dónde comienza y termina cada partición en el disco.

    Hay diferentes estándares para las tablas de particiones, siendo los más comunes el MBR (Master Boot Record) y el GPT (GUID Partition Table)

    • MBR (Master Boot Record) es un tipo de tabla de particiones y un sector de arranque que se encuentra en el primer sector (el sector cero) de un disco duro. Contiene información crítica para el inicio del sistema operativo y la organización de las particiones en el disco.
    • GPT (GUID Partition Table) es un estándar de partición más moderno y robusto. Utiliza identificadores únicos globales (GUID) para identificar particiones y proporciona una mayor flexibilidad en términos de capacidad y organización de particiones.

    5.4. Requisios Hardware

    Antes de instalar un sistema operativo en una computadora, es importante analizar ciertos requisitos de hardware para asegurarse de que el sistema operativo funcione correctamente y sea compatible con el hardware disponible. Aquí hay algunos aspectos clave que debes considerar:

    • Arquitectura del Procesador: Verifica la arquitectura del procesador (por ejemplo, x86, x64, ARM) y asegúrate de que sea compatible con el sistema operativo que deseas instalar.
    • Velocidad del Procesador: Comprueba que la velocidad del procesador cumpla con los requisitos mínimos del sistema operativo. Algunos sistemas operativos más recientes pueden requerir procesadores más potentes.
    • Memoria RAM (RAM): Asegúrate de que la cantidad de memoria RAM disponible sea suficiente para el sistema operativo. Los requisitos varían según el sistema operativo y la versión.
    • Espacio de Almacenamiento: Verifica el espacio de almacenamiento disponible en el disco duro. Asegúrate de que haya suficiente espacio para la instalación del sistema operativo y cualquier software adicional que desees instalar.
    • Tipo de Disco Duro: Algunos sistemas operativos tienen requisitos específicos en cuanto al tipo de disco duro, como SATA, SSD, o NVMe. Asegúrate de que el disco duro sea compatible.
    • Tarjeta Gráfica (GPU): Verifica la compatibilidad de la tarjeta gráfica con el sistema operativo, especialmente si estás utilizando aplicaciones o funciones que requieren capacidades gráficas específicas.
    • UEFI o BIOS: Algunos sistemas operativos modernos requieren UEFI en lugar de BIOS para el arranque. Verifica si tu sistema es compatible y ajusta la configuración del firmware según sea necesario.
    • Requisitos Específicos del Sistema Operativo: Consulta la documentación del sistema operativo para conocer requisitos específicos. Algunos sistemas operativos tienen requisitos adicionales o recomendaciones específicas.
    • Compatibilidad de 32 o 64 bits: Verifica si el sistema operativo está disponible en versiones de 32 bits o 64 bits y selecciona la versión que sea compatible con tu hardware.

    5.5. Cargaddor del S.O.

    El cargador del sistema operativo (boot loader en inglés) es un programa que se encarga de iniciar el sistema operativo cuando enciendes o reinicias tu computadora. Su función principal es cargar el núcleo (kernel) del sistema operativo en la memoria RAM para que el sistema operativo pueda comenzar a ejecutarse. Estos son los pasos que sigue el cargador del SO durante el proceso de arranque

    1. BIOS o UEFI:
      • Cuando enciendes la computadora, el proceso de arranque comienza en el firmware básico, que puede ser el BIOS (Basic Input/Output System) o el UEFI (Unified Extensible Firmware Interface).
    2. Cargador del Sistema Operativo:
      • El BIOS o UEFI carga el cargador del sistema operativo desde el MBR o la partición EFI.
    3. Selección del Sistema Operativo (Opcional):
      • Si tienes múltiples sistemas operativos instalados (arranque dual), el cargador del sistema operativo puede presentarte un menú para seleccionar el sistema operativo que deseas iniciar.
    4. Carga del Núcleo del Sistema Operativo:
      • Una vez que has seleccionado el sistema operativo, el cargador carga el núcleo (kernel) del sistema operativo en la memoria RAM.
    5. Inicio del Sistema Operativo:
      • Con el núcleo cargado en la memoria RAM, el control se transfiere al sistema operativo, y este comienza su inicialización.
      • Durante la inicialización, el sistema operativo configura los dispositivos, carga controladores, y establece el entorno de ejecución para las aplicaciones y servicios.
    6. Inicio de Sesión (Opcional):
      • En sistemas que requieren autenticación, se presenta la pantalla de inicio de sesión donde los usuarios deben ingresar sus credenciales para acceder al sistema.
    7. Interfaz de Usuario:
      • Después de la autenticación, el sistema operativo presenta la interfaz de usuario, ya sea en modo gráfico o de línea de comandos, según la configuración y el tipo de sistema operativo.

    5.6. Gestor de arranque

    El gestor de arranque (boot manager en inglés) es un programa que se encarga de facilitar la elección y el inicio de un sistema operativo específico cuando tienes varios sistemas operativos instalados en tu computadora. Su función principal es presentar al usuario una lista de sistemas operativos disponibles durante el arranque, permitiéndole seleccionar cuál desea cargar.

    Cuando tienes un sistema con múltiples sistemas operativos instalados, ya sea en una misma partición o en particiones separadas, el gestor de arranque te ofrece la capacidad de elegir cuál sistema operativo deseas utilizar en el momento de encender o reiniciar tu computadora.

    Los gestores de arranque mas populares son: - GRUB (GRand Unified Bootloader): - Utilizado comúnmente en sistemas basados en Linux, como Ubuntu, Fedora, y otras distribuciones. - Proporciona una interfaz de usuario con opciones para seleccionar el sistema operativo deseado. - Windows Boot Manager: - Utilizado en sistemas operativos Windows. - Permite seleccionar entre varias versiones de Windows instaladas en el mismo sistema.

    5.7. Fases de instalación de un S.O.

    La instalación de un sistema operativo generalmente implica varias fases, y aunque los detalles pueden variar según el sistema operativo específico, hay algunas fases comunes que suelen presentarse en la mayoría de las instalaciones. Aquí están las fases generales de instalación de un sistema operativo:

    1. Preparación:
      • Antes de comenzar la instalación, es importante realizar una copia de seguridad de los datos críticos para evitar pérdidas accidentales.
      • Verifica que el hardware de la computadora cumple con los requisitos mínimos del sistema operativo.
    2. Arranque desde el Medio de Instalación:
      • Inserta el medio de instalación, que puede ser un DVD, una unidad USB o algún otro medio específico del sistema operativo.
      • Reinicia o enciende la computadora y configura la secuencia de arranque en el BIOS o UEFI para que la computadora arranque desde el medio de instalación.
    3. Inicio del Asistente de Instalación:
      • Cuando la computadora arranca desde el medio de instalación, se inicia el asistente de instalación del sistema operativo.
      • Este asistente guiará al usuario a través de los pasos necesarios para completar la instalación.
    4. Selección del Idioma y Región:
      • El usuario selecciona el idioma y la región para la configuración regional del sistema operativo.
    5. Aceptación de Términos y Condiciones:
      • El usuario generalmente debe aceptar los términos y condiciones del acuerdo de licencia antes de continuar con la instalación.
    6. Particionamiento del Disco:
      • En esta fase, se gestionan las particiones del disco duro. Puedes optar por dejar que el sistema operativo realice un particionamiento automático o realizar una configuración manual, donde puedes crear, eliminar o modificar particiones según tus necesidades.
    7. Formato de Particiones y Copia de Archivos:
      • Las particiones seleccionadas se formatean para prepararlas para la instalación. Luego, el sistema operativo copia los archivos necesarios en las particiones correspondientes.
    8. Configuración del Sistema:
      • El usuario configura las opciones específicas del sistema operativo, como la configuración de la red, la cuenta de usuario y la contraseña, la configuración de la zona horaria, entre otros.
    9. Instalación de Controladores:
      • En algunos casos, el sistema operativo puede requerir la instalación de controladores adicionales para el hardware específico de la computadora.
    10. Finalización de la Instalación:
      • Después de completar todos los pasos, el sistema operativo realiza las configuraciones finales y notifica al usuario que la instalación ha sido exitosa.
      • Se puede requerir un reinicio para completar el proceso.
    11. Configuraciones Posteriores al Arranque:
      • Después del reinicio, es posible que se deban realizar algunas configuraciones adicionales, como la activación del sistema operativo, la instalación de actualizaciones y la configuración de preferencias adicionales.

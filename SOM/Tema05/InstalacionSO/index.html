    5. Instalación Sistemas Operativos

    5.1. Discos Duros y Particiones

    5.1.1. Discos duros

    Los discos duros son dispositivos de almacenamiento donde se guarda toda la información de un ordenador, incluyendo el sistema operativo (SO), aplicaciones y archivos del usuario. Los tipos principales son:

    • HDD: Utilizan platos magnéticos para almacenar datos, son más lentos pero tienen mayor capacidad y menor costo. Son útiles cuando se necesita gran cantidad de almacenamiento, aunque no sean los más rápidos.
    • SSD: Utilizan chips de memoria flash, lo que permite una velocidad de lectura/escritura mucho mayor. Además, al no tener partes móviles, suelen ser más resistentes y duraderos. Los SSD son ideales para sistemas operativos y programas que se benefician de un acceso rápido.Pueden ser HDD (disco duro tradicional) o SSD (unidad de estado sólido, que es más rápida).

    5.1.2. Particiones:

    Las particiones en un disco duro son divisiones lógicas o secciones separadas que se crean en el espacio de almacenamiento de un disco. Estas particiones permiten organizar y gestionar mejor la información almacenada en el disco.Hay varios tipos de particiones:

    • Partición Primaria:
      • Es la partición principal en un disco duro.
      • Puedes tener hasta cuatro particiones primarias en un disco.
      • Una de estas particiones primarias puede ser designada como "activa" y contener el cargador de arranque del sistema operativo.
    • Partición Extendida:
      • Se utiliza para crear particiones lógicas adicionales dentro de ella.
      • Puedes tener solo una partición extendida en un disco.
      • No se utiliza directamente para almacenar datos, pero sirve como contenedor para particiones lógicas.
    • Partición Lógica:
      • Se crea dentro de una partición extendida.
      • No puedes arrancar un sistema operativo directamente desde una partición lógica; necesitas una partición primaria o una partición activa para eso. -Puedes tener varias particiones lógicas dentro de una partición extendida.

    Una tabla de particiones es una estructura de datos en un disco duro que contiene información sobre la organización de las particiones en ese disco. En términos más simples, la tabla de particiones actúa como un mapa que le dice al sistema operativo dónde comienza y termina cada partición en el disco.

    Hay diferentes estándares para las tablas de particiones, siendo los más comunes el MBR (Master Boot Record) y el GPT (GUID Partition Table)

    • MBR (Master Boot Record) es un tipo de tabla de particiones y un sector de arranque que se encuentra en el primer sector (el sector cero) de un disco duro. Contiene información crítica para el inicio del sistema operativo y la organización de las particiones en el disco.
    • GPT (GUID Partition Table) es un estándar de partición más moderno y robusto. Utiliza identificadores únicos globales (GUID) para identificar particiones y proporciona una mayor flexibilidad en términos de capacidad y organización de particiones

    5.2. Requisitos de Hardware

    Antes de instalar un sistema operativo, es importante verificar que el hardware del ordenador cumpla con los requisitos mínimos del SO. Cada sistema operativo tiene requisitos específicos para funcionar adecuadamente. Estos son los componentes clave y su importancia:

    • CPU (procesador)
      • La capacidad del procesador, medida en núcleos y velocidad (GHz), influye en el rendimiento general del sistema. Por ejemplo, un procesador de doble núcleo puede realizar varias tareas a la vez, pero un procesador de cuatro núcleos suele ser más eficiente para sistemas operativos modernos y multitarea.
    • Memoria RAM
      • La RAM es esencial para la velocidad de operación. En sistemas de 32 bits, el límite de RAM suele ser de 4 GB, mientras que en sistemas de 64 bits puede llegar a ser de 8 GB o más.
      • Ejemplo de requisitos: Windows 10 necesita al menos 2 GB de RAM para la versión de 64 bits, pero se recomienda tener 4 GB o más para un rendimiento fluido.
    • Almacenamiento
      • Un sistema operativo moderno como Windows o Linux necesita entre 16 y 20 GB de espacio de almacenamiento solo para instalarse. Es recomendable dejar más espacio libre para actualizaciones y aplicaciones.
    • Tarjeta gráfica y pantalla
      • Muchos sistemas operativos requieren una tarjeta gráfica compatible para soportar gráficos de alta definición y efectos visuales, especialmente en entornos gráficos modernos como Windows y algunas distribuciones de Linux.

    Verificar los requisitos recomendados en lugar de los mínimos es ideal para garantizar un mejor rendimiento del sistema.

    5.3. Cargador del Sistema Operativo (Firmware)

    El cargador del sistema operativo es un pequeño programa que se encuentra en el disco duro y tiene la función de cargar el SO en la memoria RAM cuando se inicia el ordenador.

    Ejemplos:

    • BIOS (Basic Input/Output System):
      • Usado en equipos más antiguos. Se encarga de identificar y configurar el hardware básico del sistema y, luego, busca un sistema operativo en el disco duro.
      • Limitado en capacidades y velocidad, no soporta discos mayores a 2 TB ni el arranque rápido.
    • UEFI (Unified Extensible Firmware Interface):
      • Es la evolución del BIOS, con una interfaz gráfica y soporte para ratón. Permite arranque seguro (Secure Boot) y trabaja mejor con particiones GPT.
      • Soporta discos grandes, arranque rápido y opciones de seguridad avanzadas.

    5.4. Gestor de Arranque (Bootloader)

    El gestor de arranque es una herramienta clave cuando se instalan múltiples sistemas operativos en un mismo equipo. Su función es permitir al usuario seleccionar qué SO quiere arrancar.

    Ejemplos:

    • GRUB (GRand Unified Bootloader): Usado en Linux y compatible con Windows. GRUB detecta todos los sistemas operativos instalados y muestra un menú para elegir cuál iniciar.
    • Windows Boot Manager: El gestor de arranque predeterminado de Windows. Si solo tienes un SO de Windows, se inicia automáticamente, pero si hay otros sistemas detectados, se mostrará un menú.
    • LILO (Linux Loader): Gestor de arranque más antiguo, utilizado en distribuciones Linux anteriores a GRUB, aunque menos común en la actualidad.

    El gestor de arranque se carga justo después del BIOS o UEFI, y permite seleccionar el sistema operativo a iniciar.

    5.5. Fases de Instalación

    La instalación de un sistema operativo generalmente se lleva a cabo en las siguientes fases:

    1. Preparación del medio de instalación:
      • El sistema operativo puede instalarse desde un medio físico (USB o DVD) o mediante una descarga digital desde internet. Es importante que el medio de instalación esté en un formato que el sistema de arranque del equipo pueda reconocer (ISO).
    2. Configuración del BIOS/UEFI:
      • El usuario debe configurar el BIOS/UEFI para que el sistema arranque desde el medio de instalación, generalmente configurando la unidad USB o el lector de DVD como la primera opción de arranque.
    3. Inicio del instalador:
      • Una vez configurado el arranque, el equipo iniciará el instalador del SO, que guía al usuario a través de varios pasos de configuración inicial (como idioma, distribución de teclado y conectividad a internet).
    4. Selección y configuración de la partición:
      • Durante la instalación, el usuario selecciona en qué partición instalar el SO. Se pueden crear nuevas particiones, redimensionar las existentes o elegir una partición ya configurada. El proceso de particionado también incluye el formato del sistema de archivos, como NTFS (Windows), ext4 (Linux), o APFS (MacOS).
    5. Copia de archivos e instalación del sistema:
      • Una vez elegidas las particiones, el instalador copia los archivos del sistema operativo en el disco y, a menudo, instala controladores básicos para que los componentes del equipo (teclado, ratón, red, etc.) funcionen correctamente.
    6. Configuración inicial:
      • Se configuran los ajustes iniciales del sistema, como el nombre de usuario, idioma, la zona horaria y, en algunos sistemas, opciones de privacidad. En Windows y MacOS, se solicita una cuenta del sistema (Microsoft o Apple, respectivamente) para sincronización y actualizaciones.
    7. Instalación de actualizaciones:
      • Muchos instaladores de SO permiten descargar actualizaciones en esta etapa para que el sistema tenga las últimas mejoras de seguridad y rendimiento.
    8. Instalación de controladores adicionales:
      • En algunos sistemas, es necesario instalar controladores específicos para componentes como la tarjeta gráfica, la tarjeta de red o el sonido. Estos controladores permiten que el hardware funcione de forma óptima.
    9. Finalización e inicio del sistema operativo:
      • Al terminar el proceso de instalación, el equipo se reinicia y el gestor de arranque permite elegir el SO (si hay más de uno). Una vez seleccionado, el sistema operativo se carga y está listo para usarse.

    10. Introducción a Linux

    10.1. ¿Qué es Linux?

    • Linux es un sistema operativo de código abierto basado en Unix.
    • Fue desarrollado por Linus Torvalds en 1991.
    • Es altamente personalizable y adaptable a diferentes necesidades.
    • Se utiliza ampliamente en servidores, dispositivos embebidos, supercomputadoras y sistemas de escritorio.

    10.2. Características principales de Linux

    1. Multiusuario: Permite a múltiples usuarios trabajar en el sistema simultáneamente.
    2. Multitarea: Puede ejecutar múltiples procesos al mismo tiempo.
    3. Multiplataforma: Disponible para una amplia gama de arquitecturas de hardware.
    4. Seguridad: Ofrece un robusto sistema de permisos y control de acceso.
    5. Estabilidad: Es conocido por su estabilidad y confiabilidad, especialmente en entornos de servidor.

    10.3. Historia de Linux

    • En 1991, Linus Torvalds anunció la creación de un nuevo kernel de Unix-like.
    • La colaboración de la comunidad de desarrolladores llevó al rápido crecimiento y mejora del sistema operativo.
    • La adopción de Linux en servidores y dispositivos embebidos ha sido fundamental para su popularidad y desarrollo continuo.
    • Actualmente, Linux es parte de numerosos productos y servicios, desde servidores web hasta teléfonos móviles.

    10.4. Componentes

    En un sistema operativo Linux, varios componentes trabajan juntos para proporcionar la funcionalidad completa del sistema. Aquí hay una descripción de los componentes principales:

    • Kernel de Linux: El núcleo del sistema operativo, desarrollado por Linus Torvalds y otros colaboradores. Es responsable de administrar los recursos del hardware, como la memoria, los dispositivos de E/S, la gestión de procesos y la comunicación entre software y hardware.
    • Shell: Es la interfaz de línea de comandos entre el usuario y el núcleo del sistema. El shell interpreta los comandos ingresados por el usuario y los ejecuta. Ejemplos comunes de shells son Bash (Bourne Again Shell), Zsh (Z Shell), y Fish (Friendly Interactive Shell).
    • GNU Utilities: Una colección de herramientas y utilidades desarrolladas por el Proyecto GNU que proporcionan funcionalidades básicas del sistema, como manipulación de archivos, redirección, administración de procesos y gestión de usuarios.
    • Bibliotecas: Conjuntos de código precompilado que proporcionan funciones y rutinas comunes que los programas pueden utilizar para realizar tareas específicas. Por ejemplo, la GNU C Library (glibc) es una biblioteca comúnmente utilizada en sistemas Linux.
    • Entorno de escritorio: Un conjunto de aplicaciones y herramientas gráficas que proporcionan una interfaz de usuario amigable. Algunos ejemplos populares de entornos de escritorio para Linux incluyen GNOME, KDE, Xfce, y LXQt.
    • X Window System (X11 o Xorg): Un sistema de ventana que proporciona las bases para la interfaz gráfica de usuario en los sistemas Linux. Gestiona la representación de ventanas, gráficos y dispositivos de entrada.
    • Servicios y demonios: Procesos que se ejecutan en segundo plano y proporcionan funcionalidades específicas del sistema, como servicios de red, impresión, administración de energía, etc.
    • Sistema de archivos: Organiza y almacena los datos en el disco. Linux admite varios sistemas de archivos, como ext4, XFS, Btrfs, y más.
    • Controladores de dispositivos: Son módulos del kernel o programas que permiten al sistema operativo interactuar con hardware específico, como tarjetas de red, tarjetas gráficas, impresoras, etc.
    • Red: Linux incluye una pila de red completa que proporciona soporte para protocolos de red estándar, como TCP/IP, y permite la comunicación a través de redes locales e Internet.

    10.5. Sistema de archivos

    • En Linux, todo se representa como un archivo, incluyendo dispositivos y directorios.
    • El sistema de archivos se organiza jerárquicamente en una estructura de árbol.
    • Algunos directorios importantes incluyen /, que es el directorio raíz, /home, que contiene los directorios de usuario, y /etc, que almacena archivos de configuración.

    10.5.1. Nomenclatura de los discos

    • En Linux, los dispositivos de almacenamiento se identifican mediante un sistema de nomenclatura.
    • Por ejemplo, /dev/sda representa el primer disco duro, /dev/sdb el segundo, y así sucesivamente.
    • Las particiones en un disco se numeran secuencialmente, por ejemplo, /dev/sda1, /dev/sda2, etc.

    10.6. Gestión de paquetes

    • Los paquetes son conjuntos de software precompilado y empaquetado para su fácil instalación en Linux.
    • Los sistemas de gestión de paquetes más comunes son apt (utilizado en Debian y derivados) y yum/dnf (utilizado en distribuciones como Red Hat y Fedora).
    • Los comandos básicos para la gestión de paquetes incluyen apt install, apt remove, yum install, yum remove, entre otros.

    10.7. Proceso de instalación

    • La instalación de Linux varía según la distribución.
    • En general, implica bootear desde un medio de instalación (como un USB o un DVD) y seguir las instrucciones del instalador.
    • Durante el proceso, se puede seleccionar el idioma, la distribución de teclado, el disco de instalación, las particiones y la instalación de software adicional.
    • Después de completar la instalación, se puede configurar el sistema según las preferencias del usuario.

    10.8. Método de arranque y parada del sistema

    10.8.1. Arranque del sistema

    • El proceso de arranque del sistema en Linux se divide en varias etapas:
    • BIOS/UEFI: El firmware de la computadora realiza un autotest de hardware y busca el cargador de arranque.
    • Cargador de arranque (GRUB): Carga el kernel de Linux y la imagen del sistema de archivos inicial en la memoria.
    • Init: Inicializa el sistema y arranca los servicios esenciales.

    10.8.2. Parada del sistema

    • Para apagar o reiniciar el sistema en Linux, se puede usar el comando shutdown o reboot.
    • Por ejemplo:
    • sudo shutdown -h now: Apaga el sistema inmediatamente.
    • sudo shutdown -r now: Reinicia el sistema inmediatamente.
    • También se puede utilizar el comando halt para apagar el sistema.

    10.9. Distribuciones de Linux

    • Las distribuciones de Linux son variantes del sistema operativo que incluyen diferentes conjuntos de software y configuraciones.
    • Algunas de las distribuciones más populares son Ubuntu, Fedora, Debian, CentOS y Arch Linux.
    • Cada distribución tiene sus propias características y enfoques, lo que permite a los usuarios elegir la que mejor se adapte a sus necesidades.

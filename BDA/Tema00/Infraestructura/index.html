    0.1.2. Instalación en Fedora 40 (Portatiles del aula)

    Docker

    La documentación esta extraida de https://docs.docker.com/engine/install/fedora/

    1. Actualizar el repositorio

      sudo dnf -y install dnf-plugins-core sudo dnf-3 config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo

    2. Instalar la útima version de Docker

      sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

    3. Iniciar Docker

      sudo systemctl start docker

    4. Comprobar la instalación

      sudo docker run hello-world

    MySQL WorkBench

    La documentación esta extraida de https://computingforgeeks.com/install-and-use-mysql-workbench-on-fedora/?utm_content=cmp-true

    1. Actualizar paquetes

      sudo dnf update

    2. Descargar e intalar repositorio

      wget https://repo.mysql.com//mysql80-community-release-fc39-2.noarch.rpm sudo rpm -Uvh mysql80-community-release-*

    3. Instalar MySQL Workbench

      sudo dnf install mysql-workbench

    MongoDB Compass

    La documentación esta extraida de https://www.mongodb.com/docs/compass/current/install/

    1. Download MongoDB Compass.

      wget https://downloads.mongodb.com/compass/mongodb-compass-1.43.5.x86_64.rpm

    2. Install MongoDB Compass.

      sudo yum install mongodb-compass-1.43.5.x86_64.rpm

    3. Start MongoDB Compass.

      mongodb-compass

    En docker instalaremos los siguientes servicios - MySQL - Mongo - Nifi - Kafka

    Fichero con la configuración de MySQL + Mongo

    services:
  mongo:
    image: mongo
    container_name: mongo
    ports:
      - "28000:27017"
    volumes:
      # Sustituir /opt/data/mongo por una carpeta donde guardar las BD de mongo
      - /opt/data/mongo:/data/db
    networks:
      default:
        aliases:
          - mongo

  mysql:
    image: mysql
    container_name: mysql
    environment:
      MYSQL_ROOT_PASSWORD: tu_contraseña_mysql

          - mysql

    0.3. Docker

    Docker es una plataforma de contenedorización que permite a los desarrolladores empaquetar aplicaciones junto con todas sus dependencias en un contenedor. Esto asegura que la aplicación funcione de la misma manera en cualquier entorno, ya sea en desarrollo, pruebas o producción.

    Ventajas de Docker:

    • Portabilidad: Los contenedores pueden ejecutarse en cualquier sistema que tenga Docker instalado.
    • Eficiencia: Los contenedores son ligeros y utilizan menos recursos que las máquinas virtuales.
    • Escalabilidad: Facilita el despliegue y escalado de aplicaciones en diferentes entornos.
    • Consistencia: Asegura que las aplicaciones se ejecuten de manera idéntica en desarrollo y producción.

    0.3.1. Conceptos clave

    Contenedores: Son las unidades donde se ejecuta la aplicación, aisladas del sistema operativo host. A diferencia de las máquinas virtuales, los contenedores no necesitan un sistema operativo completo, lo que los hace más ligeros.

    Imágenes: Son plantillas de solo lectura que contienen todo lo necesario para ejecutar un contenedor: el sistema operativo, las dependencias, el código de la aplicación, entre otros. Las imágenes son versionadas y pueden descargarse desde Docker Hub (un repositorio de imágenes públicas).

    Docker Hub: Es un repositorio público donde se almacenan y distribuyen imágenes de Docker. Puedes buscar imágenes oficiales de herramientas o lenguajes como Ubuntu, Nginx, Node.js, Python, entre otros.

    Dockerfile: Es un archivo de texto que contiene una serie de instrucciones para construir una imagen de Docker. Permite definir cómo se debe construir una imagen paso a paso (instalación de dependencias, configuración del entorno, etc.).

    0.3.2. Componentes de Docker

    Docker Engine: Es el motor de Docker que se encarga de crear y gestionar contenedores. Docker CLI (Command Line Interface): Es la interfaz de línea de comandos que se utiliza para interactuar con Docker. Docker Daemon: Es el proceso que se ejecuta en segundo plano y administra los contenedores.

    0.3.3. Comandos básicos de Docker

    Confimar Instalación:

      docker --version

    Buscar una imagen en Docker Hub:

      docker search nombre_imagen

    Descargar una imagen:

      docker pull nombre_imagen

    Listar las imágenes locales:

      docker docker images

    Crear y ejecutar un contenedor: 

      docker run nombre_imagen

    Parámetro útil: -d (modo detach) para ejecutar en segundo plano.

    Listar contenedores en ejecución:

      docker ps

    Parámetro útil: -a para listar todos los contenedores, incluso los detenidos.

    Detener un contenedor:

      docker stop id_contenedor
    Iniciar un contenedor:

      docker stop id_contenedor

    Eliminar un contenedor:

      docker rm id_contenedor

    Eliminar una imagen:

      docker rmi nombre_imagen

    0.3.4. Comandos adicionales

    Acceder al contenedor

      docker exec -it nombre_contenedor bash

    Ejecutar un comando del contenedor

      docker exec -it nombre_contenedor comando

    0.3.5. Docker Compose

    Docker Compose es una herramienta que permite definir y ejecutar aplicaciones que constan de múltiples contenedores mediante un archivo docker-compose.yml.

    Ejemplo de docker-compose.yml:

    version: '3'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: postgres
    environment:
      POSTGRES_PASSWORD: ejemplo_password

    Comandos Docker Compose:

    Levantar la aplicación:

      docker-compose up

    Detener la aplicación:

      docker-compose down
